Basit Anlatım (Herkes İçin)

Ne oluyor?
Sen kodu güncelleyip "git push" dediğin an, tüm müşterilerin siteleri otomatik güncelleniyor. Gece 03:00 beklemeye gerek yok, anında!

Nasıl çalışıyor?
GitHub sana "push yaptın" diyor → Sen tuufi.com'a "yeni versiyon var" diyorsun → tuufi.com tüm müşteri sunucularına "güncellenin" diyor → Herkes git pull yapıyor.

Güvenli mi?
Evet. Her adımda secret key kontrolü var. Yetkisiz kimse güncelleme tetikleyemez.

Teknik Detaylar

1 GitHub → tuufi.com Webhook

GitHub repo ayarlarından webhook eklenir:

Webhook URL:

https://tuufi.com/api/webhook/github-deploy

Secret:

GITHUB_WEBHOOK_SECRET=xxxxx

2 tuufi.com Webhook'u Alır

Endpoint webhook'u doğrular ve işler:

// POST /api/webhook/github-deploy 1. GitHub signature doğrula (HMAC-SHA256) 2. Branch kontrol (sadece main kabul) 3. Yeni versiyon numarasını kaydet 4. Tüm aktif lisanslara bildirim gönder

3 Müşteri Sunucularına Bildirim

İki yöntem mevcut:

Yöntem A: Webhook Push

tuufi.com → Müşteri sunuculara POST atar

POST https://musteri.com/api/deploy

⚠️ Müşteri sunucu endpoint açmalı

Yöntem B: Polling ⭐

Müşteri sunucu her 1 dk kontrol eder

GET tuufi.com/api/version

✅ Daha güvenli, firewall sorunu yok

4 Müşteri Sunucu Güncellenir

Aynı v6 script'i çalışır:

git pull origin main composer install --no-dev php artisan migrate --force php artisan config:cache

Bildirim Yöntemi Seçimi

Özellik Webhook Push Polling (Önerilen) ⭐
Hız Anında (~5 sn) Max 1 dakika
Firewall Müşteri port açmalı Sorun yok (outbound)
Güvenlik Endpoint korumalı olmalı Daha güvenli
Kurulum Webhook URL gerekli Sadece cron
Sunucu Offline Webhook kaybolur Online olunca yakalar
Önerilen Kritik sistemler Genel kullanım ⭐

Öneri: Polling yöntemi daha pratik. 1 dakika gecikme çoğu senaryo için kabul edilebilir. Müşteri firewall ayarı yapmak zorunda kalmaz.

Polling Sistemi Detayı

tuufi.com (API)

GET /api/version

{
  "current_version": "2.5.1",
  "released_at": "2026-01-01 15:30:00",
  "changelog_url": "https://...",
  "critical": false
}

Müşteri Sunucu (Cron)

# Her 1 dakikada bir

* * * * * /var/www/site/check-update.sh

check-update.sh (Müşteri Sunucu)

#!/bin/bash

SITE_PATH="/var/www/site"
LICENSE_KEY="$TUUFI_LICENSE_KEY"
VERSION_FILE="$SITE_PATH/.current_version"

# Mevcut versiyonu oku
CURRENT=$(cat $VERSION_FILE 2>/dev/null || echo "0.0.0")

# Sunucudan son versiyonu al
LATEST=$(curl -s "https://tuufi.com/api/version?key=$LICENSE_KEY" | jq -r '.current_version')

# Karşılaştır
if [ "$CURRENT" != "$LATEST" ]; then
    echo "Yeni versiyon: $LATEST (mevcut: $CURRENT)"

    # Güncelleme script'ini çalıştır
    $SITE_PATH/update.sh

    # Yeni versiyonu kaydet
    echo "$LATEST" > $VERSION_FILE
fi

Güvenlik Önlemleri

GitHub Webhook Secret

HMAC-SHA256 ile imza doğrulama. Sahte webhook engelenir.

License Key Kontrolü

Her API isteğinde lisans anahtarı doğrulanır.

Branch Kontrolü

Sadece main branch push'ları kabul edilir.

Rate Limiting

Dakikada max 1 güncelleme isteği kabul edilir.
