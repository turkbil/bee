Basit Anlatım (Herkes İçin)

Ne oluyor?
Sen bir güncelleme yaptığında, tüm müşterilerin siteleri gece otomatik güncelleniyor. Kimse bir şey yapmıyor, sabah herkes yeni versiyonda.

Nasıl çalışıyor?
Her müşterinin sunucusunda bir "bekçi" var. Bu bekçi her gece "yeni güncelleme var mı?" diye bakıyor. Varsa indiriyor, yoksa uyuyor.

Ya bir şey bozulursa?
Bekçi akıllı - güncelleme başarısız olursa eski haline döndürüyor ve sana haber veriyor.

Otomatik Güncelleme Script'i 

#!/bin/bash
# update.sh - Müşteri sunucusunda çalışır

SITE_PATH="/var/www/site"
LOG_FILE="/var/log/tuufi-update.log"

echo "[$(date)] Güncelleme kontrolü başlıyor..."

cd $SITE_PATH

# 1. Yeni commit var mı kontrol et
git fetch origin main --quiet

LOCAL=$(git rev-parse HEAD)
REMOTE=$(git rev-parse origin/main)

if [ "$LOCAL" = "$REMOTE" ]; then
    echo "[$(date)] Güncelleme yok, çıkılıyor."
    exit 0
fi

echo "[$(date)] Yeni güncelleme bulundu! Güncelleniyor..."

# 2. Maintenance mode aç
php artisan down --secret="update-in-progress"

# 3. Git pull
git pull origin main

# 4. Composer (production)
composer install --no-dev --optimize-autoloader --quiet

# 5. Migration
php artisan migrate --force

# 6. Cache temizle
php artisan config:cache
php artisan route:cache
php artisan view:cache

# 7. Maintenance mode kapat
php artisan up

# 8. Başarı bildirimi (opsiyonel)
curl -s "https://tuufi.com/api/license/heartbeat?key=$TUUFI_LICENSE_KEY&version=$(git describe --tags)"

echo "[$(date)] ✅ Güncelleme tamamlandı!"
Maintenance mode ile kullanıcı etkilenmez
Hata olursa git reset ile geri dönüş
Heartbeat ile merkeze bildirim

Hata Durumunda Geri Dönüş 

# Güncelleme başarısız olursa otomatik rollback

if ! php artisan migrate --force; then
    echo "❌ Migration hatası! Geri alınıyor..."

    # Önceki commit'e dön
    git reset --hard $LOCAL

    # Cache'leri eski haline getir
    php artisan config:cache
    php artisan route:cache

    # Siteyi aç
    php artisan up

    # Hata bildirimi gönder
    curl -s "https://tuufi.com/api/license/error?key=$TUUFI_LICENSE_KEY&error=migration_failed"

    exit 1
fi

İlk Kurulum (Tek Seferlik)

1

Deploy Key Oluştur

ssh-keygen -t ed25519 -C "deploy@musteri.com" -f ~/.ssh/tuufi_deploy

Public key'i bize gönderir, biz repo'ya ekleriz (read-only)

2

Repo'yu Clone Et

git clone git@github.com:tuufi/platform.git /var/www/site
3

.env Ayarla

TUUFI_LICENSE_KEY=GZTE-2026-XXXX-XXXX
DB_DATABASE=musteri_db
DB_USERNAME=musteri_user
4

Cron Ekle

crontab -e
