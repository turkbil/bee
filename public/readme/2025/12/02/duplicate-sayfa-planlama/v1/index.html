Kategori Analizi

1. Search Sayfalari 3 sayfa

Arama sonuc sayfalari - dinamik icerik, duplicate riski yuksek.

/search/IXTIF F4 1350x685 mm Catal /search/IXTIF F4 201 1150x560 mm Catal /search/IXTIF CPD18FVL 1070 mm Catal Forklift

Cozum Onerisi

  • Sonucu olan aramalar: index, follow (kullaniciya deger katiyor)
  • Sonucu olmayan aramalar: noindex, follow
  • /search/tags: index, follow (benzersiz icerik)
  • robots.txt'de /search/ bloklama KALDIRILMALI (canonical + noindex yeterli)
2. Blog Yazilari (AI Uretilmis) ~45 sayfa

AI ile uretilmis blog icerikleri. Google bunlari "thin content" veya "duplicate" olarak isaretlemis.

/transpalet-nedir-kullanim-alanlari-ve-avantajlari /forklift-modelleri-hangi-modeli-secmelisiniz /elektrikli-transpalet-mi-manuel-transpalet-mi /kiralik-forklift-fiyatlari-2025-ne-beklemelisiniz ... ve 40+ daha

Karar Gerekiyor

Bu bloglar neden duplicate isaretlendi?

  • Ihtimal 1: Icerik internetteki diger sitelerle cok benzer (AI ayni seyleri uretiyor)
  • Ihtimal 2: Icerik cok kisa/sIG (thin content)
  • Ihtimal 3: Meta description'da JSON gorunuyor (teknik hata - bunu duzeltmek lazim!)
  • Ihtimal 4: Birden fazla URL ayni icerigi gosteriyor

Cozum Secenekleri

  • Secenek A: Icerikleri zenginlestir - daha uzun, daha ozgun, gorseller ekle
  • Secenek B: Benzer konulari birlestir (ornek: 5 transpalet yazisi -> 1 kapsamli rehber)
  • Secenek C: Dusuk kaliteli olanlari noindex yap veya sil
  • Secenek D: Meta description JSON hatasini duzelt (oncelikli!)
3. Statik Sayfalar 5-6 sayfa

Site'nin temel sayfalari - bunlarin duplicate olmamasi gerekir!

Inceleme Gerekiyor

  • Bu sayfalar neden duplicate? Canonical dogru mu?
  • Ayni icerigin farkli URL'leri var mi?
  • Icerik yeterince ozgun mu?

Tespit Edilen Teknik Hata

Meta Description'da JSON Gorunuyor! Kritik
<meta name="description" content="```json { "title": "Transpaletlerin..."

Blog sayfalarinin meta description'inda JSON kodu gorunuyor. Bu buyuk bir SEO hatasi - Google bu sayfalari dusuk kaliteli olarak isaretler.

Acil Duzeltme Gerekiyor

Blog meta description'lari duzgun metin olmali, JSON degil!

Onerilen Eylem Plani

1

Meta Description JSON Hatasini Duzelt

Blog sayfalarinda description'a JSON yerine duzgun metin yazilmali. Bu en acil sorun.

2

Search Sayfalari icin Akilli noindex

Sonuc varsa index, yoksa noindex. /search/tags her zaman index.

3

robots.txt'den /search/ Kaldir

Canonical + noindex yeterli. robots.txt engellemesi gereksiz ve SEO'ya zarar verebilir.

4

Statik Sayfalari Incele

Hakkimizda, Hizmetler, SSS sayfalarinin canonical'larini ve iceriklerini kontrol et.

5

Blog Icerik Stratejisi Belirle

Kullanici karari: Icerikleri zenginlestir mi, birlestir mi, sil mi?
