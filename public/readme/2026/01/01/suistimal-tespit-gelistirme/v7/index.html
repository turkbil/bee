Döngü Bazlı Tespit

Hesap Paylaşımı Tespiti

Önemli olan tek seferlik değişim değil, DÖNGÜ!

Temel Mantık

1 Üye = 1 Cihaz = 1 IP (aynı anda)

Ama kullanıcı lokasyon değiştirebilir. Önemli olan kalıcılık.

NORMAL

Biri durur, diğeri başlar ve KALICI

Hafta 1: IP-A (ofis)

Hafta 2: IP-B (yeni ofis)

→ Taşınmış, sorun yok

SUİSTİMAL

Sürekli gidip gelme DÖNGÜ

Pazartesi: IP-A → IP-B → IP-A

Salı: IP-B → IP-A → IP-B

→ Ping-pong = paylaşım

Tespit Kuralları

1

Eş Zamanlı Farklı IP/Device

Aynı anda 2 farklı yerden dinleme = KESİN SUİSTİMAL

// Senaryo: Overlap + Farklı IP

10:00-10:05 Şarkı A | IP: 92.44.x

10:02-10:07 Şarkı B | IP: 88.230.x

→ 2 farklı lokasyonda aynı anda = %100 paylaşım

2

IP/Device Döngüsü (Ping-Pong)

Biri durur diğeri başlar, sonra tekrar eski başlar = SUİSTİMAL

// Senaryo: Sıralı ama döngü var

09:00 IP-A başlar

12:00 IP-A durur → IP-B başlar

15:00 IP-B durur → IP-A başlar

18:00 IP-A durur → IP-B başlar

→ A↔B döngüsü = 2 kişi sırayla kullanıyor

Tespit: X aralıkta (örn: 7 gün) IP-A ve IP-B arası 3+ geçiş varsa = döngü
3

Tek Seferlik Geçiş (Normal)

Biri durur, diğeri başlar ve KALICI kalır = Normal

// Senaryo: Lokasyon değişikliği

1-15 Ocak: IP-A (eski ofis)

16 Ocak: Geçiş

17-31 Ocak: IP-B (yeni ofis)

→ Kalıcı geçiş = işletme taşınmış, sorun yok

Döngü Tespiti Nasıl Yapılır?

Algoritma

1. Fingerprint Geçişlerini Listele

Her play kaydında önceki play'in fingerprint'i ile karşılaştır.

geçişler = []
for i = 1 to plays.length:
  if plays[i].fingerprint != plays[i-1].fingerprint:
    geçişler.push({from: plays[i-1].fingerprint, to: plays[i].fingerprint})

2. Geri Dönüş Kontrol Et

Bir fingerprint'e geri dönüş var mı?

A → B (geçiş 1)
B → A (geçiş 2) ← GERİ DÖNÜŞ!
A → B (geçiş 3) ← TEKRAR!

3. Döngü Sayısını Hesapla

X gün içinde kaç kez geri dönüş var?

0-1 Geri Dönüş

Normal

2 Geri Dönüş

Şüpheli
