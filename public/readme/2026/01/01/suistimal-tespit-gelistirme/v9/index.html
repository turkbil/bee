Suistimal Tespiti

1 Abonelik = 1 Cihaz = 1 Yayın

Temel Kural

1 Abonelik = 1 Aktif Yayın

2 hoparlöre ayrı yayın = 2 abonelik gerektirir

Tespit Edilecek Suistimal Türleri

1. Ping-Pong Pattern

Herhangi bir veride A → B → A döngüsü = Hesap paylaşımı

Kontrol Edilecek Veriler:

  • • IP adresi
  • • Browser (Chrome, Safari...)
  • • Platform (Windows, macOS...)
  • • Device (Desktop, Mobile)

Örnek:

Chrome → Safari → Chrome
Windows → macOS → Windows
IP-A → IP-B → IP-A

= SUİSTİMAL

Not: IP dinamik olabilir. Tek başına IP değişimi yeterli değil, döngü olmalı.

2. Eş Zamanlı Farklı Kaynak

Aynı anda 2 farklı IP/device/browser = Kesin suistimal

Örnek:

10:00-10:05 Şarkı A | IP: 92.44.x | Chrome
10:02-10:07 Şarkı B | IP: 88.230.x | Safari

= Aynı anda 2 farklı lokasyon = SUİSTİMAL

3. Çoklu Yayın (Split Stream)

Aynı PC'den özel yazılımla 2 ayrı hoparlöre yayın

Nasıl Çalışıyor?

1 PC

Özel Yazılım

Hoparlör 1

Hoparlör 2

Tespit Nasıl?

Aynı fingerprint (IP + Browser + Platform) ama:

  • • Şarkılar aynı anda bitiyor (senkronize)
  • • Süreler çok yakın (16, 17, 18, 18, 20, 21 dk gibi)
  • • Next tuşu hiç kullanılmamış
  • • Her şarkı tam sonuna kadar gitmiş
  • • 2-3 dk'lık şarkılarda bile tutarlı pattern

Örnek Timeline:

Stream 1: Şarkı A (3:24) → Şarkı B (2:58) → Şarkı C (4:12)

Stream 2: Şarkı X (3:24) → Şarkı Y (2:58) → Şarkı Z (4:12)

→ Aynı süreler, aynı pattern = 2 paralel yayın!

Çoklu Yayın Tespiti (Detay)

Tespit Kriterleri

1. Aynı Fingerprint'ten Çok Fazla Play

Normal: Günde 300-400 şarkı (tek hoparlör, 10-12 saat)

Şüpheli: Günde 600-800+ şarkı (2 hoparlör, paralel yayın)

2. Şarkı Sürelerinin Tutarlılığı

Her şarkı tam bitiyor + aralar çok tutarlı:

listened_duration = song.duration (her seferinde)

gap_between_songs = 0-2 saniye (sürekli aynı)

3. Next/Skip Kullanımı = 0

İnsan dinlerken bazen skip eder, bazen yarıda bırakır.

Otomatik yayın: Hiç müdahale yok, playlist olduğu gibi çalıyor.

4. Overlap (Aynı Fingerprint)

Aynı IP/browser/platform'dan aynı anda 2 şarkı çalıyorsa:

→ 2 sekme/pencere açık = 2 hoparlöre yayın

Sonuç:

Bu pattern görüldüğünde: 1 PC ile 2 farklı yere yayın yapılıyor.

1 abonelik = 1 yayın. 2 hoparlör = 2 abonelik gerekli!

Özet

Pattern Tespit Anlam Sonuç
Ping-Pong A → B → A döngüsü 2 kişi sırayla kullanıyor SUİSTİMAL
Eş Zamanlı Farklı Aynı anda 2 farklı IP/device 2 farklı lokasyon SUİSTİMAL
Çoklu Yayın Aynı fingerprint + overlap veya çok yüksek hacim + tutarlı süreler 1 PC → 2 hoparlör SUİSTİMAL
Normal Tek fingerprint, ara sıra skip, değişken aralar Normal kullanım TEMİZ

Implementasyon

detectPingPong($field)

IP, browser, platform, device için A→B→A pattern kontrolü

detectConcurrentDifferentSource()

Aynı anda farklı fingerprint'lerden yayın var mı?

detectSplitStream()

Aynı fingerprint + (overlap VEYA yüksek hacim + tutarlı süreler + skip yok)
