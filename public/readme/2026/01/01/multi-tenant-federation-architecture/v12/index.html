1 Problem ve Mevcut Durum

Basit Anlatim (Herkes Icin)

Tuufi.com bir SaaS platformu - yani musterilere hazir web sitesi/uygulama sunuyor. Kucuk musteriler tuufi.com uzerinde paylasiyor (shared). Ama buyuk musteriler (kurumsal firmalar) kendi sunucularinda calistirmak istiyor (self-hosted).

Problem: Kendi sunucusuna kuran musteri, lisans olmadan sistemi kullanabilir mi? Kodlari kopyalayabilir mi? Guncellemeleri nasil alacak? 100+ musteri varsa her birine manuel mi gidecegiz?

Teknik Detaylar (Gelistiriciler Icin)

  • Framework: Laravel 11 + Stancl/Tenancy v3.8
  • Mevcut Yapi: Tek sunucu, tum tenant'lar ayni DB cluster
  • Sorun 1: Self-hosted deployment'ta lisans kontrolu yok
  • Sorun 2: Kod acik, musteri isterse kopyalar
  • Sorun 3: 100+ musteriye manuel guncelleme = imkansiz
  • Sorun 4: License server down = tum sistem durur (SPOF)

2 Musteri Tipleri

SHARED (Paylasimli)

  • Nedir: Kucuk siteler, tuufi.com uzerinde
  • Ornek: KOBƒ∞ web siteleri, portfolyolar
  • Altyapi: Ayni sunucu, ayni DB cluster
  • Guncelleme: Otomatik (tek git pull)
  • Lisans: Gerekli degil (zaten bizim sunucumuz)
  • Fiyat: Aylik 50-200 TL
Simdi: ~50 tenant aktif

SELF-HOSTED (Kendi Sunucusu)

  • Nedir: Buyuk musteriler, kendi sunucularinda
  • Ornek: Kurumsal firmalar, bankalar, kamu
  • Altyapi: Musterinin kendi sunucusu
  • Guncelleme: Webhook + Polling ile otomatik
  • Lisans: ZORUNLU (koruma + kontrol)
  • Fiyat: Yillik 10.000-50.000 TL
Hedef: 100+ kurumsal musteri

Lisans dosyasi dijital imzali, degistirilirse bozulur

7 Sifreli Cekirdek Dosya (core.enc) v10'dan

Basit Anlatim

Sistemin en kritik kodu core.enc dosyasinda. Bu dosya AES-256 ile sifrelenmis. Iceriginde lisans kontrolu, guncelleme mantigi, guvenlik kontrolleri var.

1 byte bile degistirirsen - hash uyusmaz, sistem calismaz.
Dosyayi silersen - sistem calismaz.
Sahte dosya koyarsan - decrypt basarisiz, sistem calismaz.

core.enc Dosya Yapisi

// Dosya Konumu:
app/Core/core.enc
// Icerik (sifrelenmis):
- LicenseValidator class
- UpdateManager class
- SecurityChecker class
- FeatureFlagManager class
// Sifreleme:
AES-256-CBC + HMAC-SHA256

CoreLoader Mekanizmasi (5 Adim)

1
Dosya Kontrolu

core.enc var mi? Boyutu dogru mu?

2
Hash Dogrulama

SHA-256 hash'i license sunucusundan al, karsilastir

3
Decrypt

AES-256 ile sifre coz (key: license_key + hardware_id)

4
Eval/Include

Cozulen PHP kodunu calistir

5
Memory Temizligi

Plain-text kodu bellekten sil, cache'e YAZMA

4 Koruma Senaryosu

1. Dosya Degistirildi

  • ‚Üí Hash uyusmaz
  • ‚Üí Sistem DURUR
  • ‚Üí Admin'e bildirim gider

2. Dosya Silindi

  • ‚Üí CoreLoader "dosya yok" hatasi
  • ‚Üí Sistem DURUR
  • ‚Üí Kurulum gerekli

3. Bypass Denemesi

  • ‚Üí Coklu kontrol noktasi var
  • ‚Üí Birini bypass etsen digerleri yakalar
  • ‚Üí Sistem DURUR + alarm

4. Sahte Dosya

  • ‚Üí Decrypt basarisiz (yanlis key)
  • ‚Üí Garbage output
  • ‚Üí PHP syntax error, sistem DURUR

Offline Calisma

24 Saat Cache:

core.enc hash'i 24 saat cache'lenir. License sunucusuna her seferinde gitmez.

7 Gun Grace Period:

License sunucusu 7 gun down olsa bile sistem calisir. 8. gun DURUR.

10 Guncelleme Sistemi

Basit Anlatim

100+ musteriye tek tek gidip "guncelle" diyemeyiz. Sistem otomatik olmali.
Webhook: Biz kod push edince, tum musterilere "guncelleme var" diye sinyal gider.
Polling: Musteri her 1 dakikada "guncelleme var mi?" diye sorar (yedek yontem).

Guncelleme Akisi

1
Gelistirici: git push (tuufi.com'a)
2
tuufi.com: Yeni versiyon algilandi, core.enc yeniden olustur
3
Webhook: Tum self-hosted musterilere "guncelleme var" sinyali
4
Musteri Sunucusu: Sinyal al, git pull yap, core.enc guncelle
5
Sonuc: ~1 dakika icinde tum musteriler guncel!

11 Webhook Sistemi

Basit Anlatim

Webhook = "Bir sey olunca bana haber ver". Biz kod push ettigimizde, sistem otomatik olarak tum musterilere HTTP request atar. Musteri bu request'i alinca kendini gunceller.

Webhook Event Tipleri

update.available

Yeni versiyon var, guncelleme yap

license.renewed

Lisans yenilendi, cache'i temizle

license.revoked

Lisans iptal edildi, sistemi durdur

feature.toggled

Bir ozellik acildi/kapandi

Webhook Guvenlik

  • HMAC-SHA256 Imza: Her webhook request'i imzali, sahte request kabul edilmez
  • Timestamp Kontrolu: 5 dakikadan eski request'ler reddedilir (replay attack korumasƒ±)
  • IP Whitelist: Sadece tuufi.com IP'lerinden gelen request kabul

13 API Dokumantasyonu

Basit Anlatim

Musteri sunucusu bu API'leri kullanarak lisans sunucusuyla iletisim kurar. Aktivasyon, dogrulama, heartbeat, guncelleme kontrolu hep bu API'ler uzerinden.

POST /api/license/activate

Ilk kurulumda lisansi aktive et

Params: license_key, domain, hardware_id
POST /api/license/verify

Lisansin gecerli olup olmadigini kontrol et

Params: license_key, domain, hardware_id
POST /api/license/heartbeat

Her 6 saatte "yasiyorum" sinyali

Params: license_key, hardware_id, stats (opsiyonel)
POST /api/license/challenge v11'den

Challenge-response dogrulama

Params: license_key, challenge_token, response
GET /api/core/hash v10'dan

Guncel core.enc hash'ini al (dogrulama icin)

Params: license_key
GET /api/update/check

Yeni versiyon var mi kontrol et

Params: license_key, current_version
GET /api/update/download

Guncelleme paketini indir

Params: license_key, version
