Problem Tanimi

Basit Anlatim

Tum musteriler ayni sunucuda. Sunucumuz kucuk. Buyuk musteriler (gazeteler, hastaneler) kendi sunucularini kullanmak istiyor. Hem kucuklere ortak hizmet, hem buyuklere kendi sunucularinda calisma imkani vermemiz lazim. Ayrica ana sunucu cokerse digerleri etkilenmemeli.

Teknik Detaylar

Mevcut Sorunlar:

  • * Tek sunucu = Tek hata noktasi (SPOF)
  • * Sunucu kapasitesi sinirli
  • * Buyuk musteriler icin yeterli kaynak yok
  • * Ana sunucu cokerse herkes etkileniyor

Hedefler:

  • * Farkli musteri tiplerine farkli cozum
  • * Self-hosted musteri destegi
  • * Ayri License Server (SPOF cozumu)
  • * Sifreli cekirdek dosya (kirilmaz koruma)

Sifreli Cekirdek Dosya

YENI - v10

Basit Anlatim (Herkes Icin)

Sistemin calismasi icin ZORUNLU olan sifreli bir dosyamiz olacak. Bu dosya kasa gibi - icinde kritik bilgiler var ve acilmasi icin anahtar lazim. Anahtar sadece lisans sunucusundan geliyor.

Neden Kirilmaz?

  • Dosya AES-256 ile sifrelenmis (askeri seviye)
  • 1 harf bile degisse dosya acilamaz
  • Sifre cozme anahtari bizde, dosya musteride
  • Anahtar olmadan sistem calismaz

Teknik Detaylar (Gelistiriciler Icin)

Sifreli Dosya Icerigi:

// core.enc (sifreli)
{
    "app_key": "base64:xxxxx",
    "license_validator": "class hash",
    "critical_routes": [...],
    "feature_flags": {...},
    "encryption_salt": "xxxxx",
    "signature": "sha256:xxxxx"
}

Dosya Konumu:

/app
  /Core
    core.enc          <- Sifreli dosya
    CoreLoader.php    <- Yukleyici

/bootstrap
  app.php             <- Sistem baslarken
                         CoreLoader cagirilir

Nasil Calisir?

1

Sistem Baslarken

Laravel boot olurken CoreLoader::init() cagirilir

bootstrap/app.php -> CoreLoader::init()
2

core.enc Dosyasini Oku

Sifreli dosya okunur, hash'i alinir

$encrypted = file_get_contents('core.enc');
$hash = sha256($encrypted);
3

Lisans Sunucusuna Sor

Hash + lisans key gonderilir, desifre anahtari istenir

POST license.tuufi.com/api/core/decrypt
{
    "license_key": "TUUFI-XXX",
    "file_hash": "sha256:abc123..."
}
4

Sunucu Kontrol Eder

Lisans + hash dogruysa desifre anahtari doner

// Basarili yanit
{
    "valid": true,
    "decrypt_key": "aes256:xyz789...",
    "expires_in": 86400  // 24 saat
}
5

Dosya Acilir, Sistem Calisir

Anahtar ile dosya desifre edilir, kritik veriler yuklenir

$data = decrypt($encrypted, $key);
Config::set('app.key', $data->app_key);

Koruma Senaryolari

Senaryo: Dosyayi Degistirdiler

core.enc dosyasinda 1 harf bile degisse...

1. Hash degisir -> Sunucudaki hash ile eslesmiyor
2. Sunucu "invalid hash" der -> Anahtar vermez
3. Dosya acilamaz -> SISTEM CALISMAZ

Senaryo: Dosyayi Sildiler

core.enc dosyasini silerlerse...

1. CoreLoader dosyayi bulamaz
2. Kritik config degerleri yuklenmez
3. App key yok -> LARAVEL BOOT OLAMAZ

Senaryo: CoreLoader'i Bypass Ettiler

CoreLoader cagrisini kaldirdılar...

1. app.key yuklenmez -> Session/encryption calismaz
2. critical_routes yuklenmez -> Admin panel acilmaz
3. feature_flags yuklenmez -> OZELLIKLER CALISMAZ

Senaryo: Sahte core.enc Olusturdular

Kendi core.enc dosyalarini yazdılar...

1. Hash bizim sunucudakiyle eslesmiyor
2. Sunucu anahtar vermez
3. ACIK ANAHTARI BILMIYORLAR -> IMKANSIZ

Offline Calisma (Grace Period)

Internet kesilirse sistem durmasin diye desifre anahtari 24 saat gecerli. Cache'e kaydedilir. 7 gun icerisinde en az 1 kez sunucuya ulasilmali. 

// Anahtar cache'lenir
Cache::put('core_decrypt_key', $key, 86400); // 24 saat

// Her request'te kontrol
if (Cache::has('core_decrypt_key')) {
    // Cache'den kullan
} else {
    // Sunucudan yenile
}

Git Push Sonrasi Akis

// 1. Biz yeni versiyon push ettik
git push origin main

// 2. Yeni core.enc olusturulur (biz yapariz)
php artisan core:encrypt --version=2.1.0
-> Yeni core.enc + hash olusturuldu

// 3. Hash lisans sunucusuna kaydedilir
POST license.tuufi.com/api/core/register-hash
{
    "version": "2.1.0",
    "hash": "sha256:newHash123..."
}

// 4. Musteriler git pull yapar
-> Yeni core.enc indirilir

// 5. Sistem baslarken yeni hash ile anahtar ister
-> Sunucu yeni hash'i tanir, anahtar verir
-> Sistem calisir!

Diger Koruma Katmanlari

Katmanli Koruma Sistemi

1

Sifreli Cekirdek Dosya (core.enc)

En guclu koruma. Dosya degisirse sistem calismaz. 1 byte = fail.

YENI - v10
2

Coklu Kontrol Noktasi

Middleware, ServiceProvider, Scheduler, Controller - birini bypass etsen digeri yakalar.

3

Kod Butunlugu Kontrolu (Checksum)

Kritik PHP dosyalarinin hash'i sunucuda saklanir. Degistirilirse alarm.

4

Kritik Veri Sunucudan Gelir

Sifreleme anahtarlari, premium ozellik flag'leri sunucudan alinir.

5

Anomali Tespiti

Heartbeat durmussa, farkli IP'den istek geliyorsa uyari.

Koruma Ozeti

Bypass Girisimleri:

  • X core.enc sil -> Laravel boot olamaz
  • X core.enc degistir -> Hash uyusmaz
  • X CoreLoader bypass -> Config yuklenmez
  • X Lisans kontrolu sil -> Coklu checkpoint yakalar
  • X Offline calis -> 7 gun sonra durur

Sonuc:

  • + core.enc = Kilit
  • + Sunucudaki anahtar = Tek anahtar
  • + Hash kontrolu = Degisiklik tespiti
  • + Coklu checkpoint = Yedek koruma
  • = BYPASS IMKANSIZ
