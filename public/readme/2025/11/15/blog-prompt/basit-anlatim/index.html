2 Aşamalı Prompt Sistemi

Blog üretim sistemi 2 aşamalı çalışır: Taslak → İçerik

AŞAMA 1: Blog Taslağı Oluştur

İçerik yapısını planla, SEO stratejisini belirle

AI Prompt İstekleri:

1. SEO Meta Bilgileri
  • • Title (50-60 karakter, anahtar kelime başta)
  • • Meta description (155-160 karakter, CTA ile)
  • • URL slug + Focus keyword
2. Schema.org Planı
  • • Article Schema (zorunlu)
  • • FAQPage Schema (5-10 soru)
  • • BreadcrumbList Schema
  • • Product/HowTo Schema (içeriğe göre)
3. İçerik Yapısı (2000-2500 kelime)
  • • H1 başlık
  • • 4-6 H2 başlık + H3 alt başlıklar
  • • Her bölüm için kelime sayısı
  • • Anahtar kelime dağılımı
  • • Görsel/tablo/liste gereksinimleri
  • • Dahili link fırsatları
4. FAQ Soruları (10 adet)
  • • "X nedir?" formatında
  • • "Nasıl...?" formatında
  • • "Hangisi daha iyi?" formatında
  • • Long-tail anahtar kelimeler

AŞAMA 2: Blog İçeriğini Yaz

Taslağa göre bölüm bölüm blog içeriğini oluştur

Yazım Kuralları:

Hedef Kitle (B2B)
  • • 25-65 yaş
  • • Depo yöneticileri
  • • Satın alma müdürleri
  • • Lojistik sorumlular
  • • Teknik ekipler
Ton & Stil
  • • Profesyonel, teknik
  • • Sade, kesin, net
  • • Cümle ≤ 20 kelime
  • • Paragraf ≤ 150 kelime
  • • Marka adı kullanma
SEO Kuralları
  • • Keyword density: %1-2
  • • LSI terimleri kullan
  • • TF-IDF analizi
  • • Featured snippet optimize
  • • 5-10 dahili link
Kaynaklar
  • • ISO standartları
  • • Teknik dökümanlar
  • • Sektör otoriteleri
  • • Akademik yayınlar
  • • Her iddia için kanıt

Kredi Sistemi - Mevcut Altyapı

Sistem zaten hazır! Yeni migration gerekmiyor.

Mevcut Altyapı

Tenant Model:

  • ✅ tenants.ai_credits_balance
  • ✅ hasEnoughCredits()
  • ✅ useCredits()

AI Modülü:

  • ✅ ai_use_credits()
  • ✅ ai_can_use_credits()
  • ✅ AICreditUsage model

Kredi Maliyeti

1.0
Araştırma (toplam)
100 taslak da olsa
1.0
Blog başına
2000-2500 kelime

Örnek Hesaplama:

  • • Araştırma (100 taslak): 1.0 kredi
  • • 10 Blog yazımı: 10.0 kredi (1.0 × 10)
  • • TOPLAM: 11.0 kredi

Kod Örneği 

use function ai_can_use_credits;
use function ai_use_credits;

// 1. Kredi kontrolü
if (!ai_can_use_credits(0.05)) {
    throw new InsufficientCreditException('Kredi yetersiz!');
}

// 2. Blog oluştur
$blog = $this->generateBlogContent($topic, $category);

// 3. Kredi düş
ai_use_credits(0.05, null, [
    'usage_type' => 'blog_generation',
    'operation_type' => 'content_generation',
    'word_count' => str_word_count($blog->content),
    'reference_id' => $blog->id
]);

// 4. Blog kaydet
$blog->save();

Tenant-Specific Prompt Customization

Her tenant için farklı AI prompt ve context kullanabilme sistemi. ixtif.com shop odaklı, başka tenant farklı sektör - her biri özel blog üretir!

Klasör Yapısı 

Modules/Blog/app/Services/TenantPrompts/
├── TenantPromptLoader.php       # Ana loader servisi
├── DefaultPrompts.php           # Default prompt'lar (fallback)
└── Tenants/
    ├── Tenant2Prompts.php       # ixtif.com (shop odaklı)
    ├── Tenant3Prompts.php       # Gelecekteki tenant'lar
    └── ...

TenantPromptLoader

  • Tenant ID'yi al
  • Özel dosya var mı kontrol et
  • Varsa tenant prompt kullan
  • Yoksa default kullan (güvenli)

Tenant2Prompts (ixtif.com)

  • Shop modülü odaklı
  • Forklift, Transpalet
  • Referanslar
  • Hizmetler

DefaultPrompts

  • Güvenli fallback
  • Genel prompt
  • Setting Group 6,10
  • Yeni tenant'lar için

Avantajlar

  • Tenant 2 (ixtif): Shop, ürünler odaklı
  • Tenant 3: Farklı sektör → Farklı prompt
  • Yeni tenant: Default kullanır, sorunsuz
  • Özelleştirme: Kod değişikliği gerektirmez

Örnek Senaryolar

  • Shop var: Ürün bazlı blog
  • Shop yok: Genel içerik
  • Referanslar var: Müşteri hikayeleri
  • Hizmetler var: Hizmet odaklı
