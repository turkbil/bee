KARAR #1: Database Stratejisi

Media Tablosu mu? Yoksa Eski Usul Kendi Tabloları mı?

ESKİ USUL DEVAM! String path kullanacağız (muzibu_songs tablosu)
Neden Media Tablosu Değil?
Özellik Media Tablosu (Spatie) String Path (Eski Usul)
25,000 Eski Şarkı Hepsini migrate etmek zor Sadece path güncelle
Basitlik Polymorphic ilişkiler karmaşık Tek field: file_path
Hız Join, ilişki query'leri Direkt path oku
Özel Metadata Custom property desteği JSON field ekleyebiliriz
Sonuç
String path yöntemi bu projede daha mantıklı çünkü:
  • 25,000 eski şarkı var - hepsini migrate etmek gereksiz iş
  • Basit ve hızlı çözüm
  • Thumbmaker zaten string path ile çalışıyor
  • HLS conversion path'leri için de uygun
Database Şeması
// muzibu_songs tablosu (eski fieldlara ek olarak) ALTER TABLE muzibu_songs ADD COLUMN file_path VARCHAR(500); ALTER TABLE muzibu_songs ADD COLUMN hls_path VARCHAR(500) NULL; ALTER TABLE muzibu_songs ADD COLUMN hls_converted TINYINT(1) DEFAULT 0; ALTER TABLE muzibu_songs ADD COLUMN duration INT NULL -- saniye cinsinden; ALTER TABLE muzibu_songs ADD COLUMN bitrate INT NULL -- kbps; ALTER TABLE muzibu_songs ADD COLUMN metadata JSON NULL -- ekstra bilgiler; // Görseller için (zaten var ama örnekleme) muzibu_albums.thumb = 'muzibu/images/albums/album-123.jpg' muzibu_artists.thumb = 'muzibu/images/artists/artist-456.jpg'

KARAR #3: Migration Planı (25,000 Şarkı)

Eski 25,000 Şarkıyı Nasıl Migrate Edeceğiz?

1
Eski Şarkıları Yeni Dizine Kopyala

Eski sistemdeki şarkılar neredeyse, onları yeni dizin yapısına kopyalayacağız.

# Örnek: Eski şarkılar /old/muzik/sarkılar/ klasöründeyse cp -r /old/muzik/sarkılar/* storage/app/public/muzibu/songs/original/ # Veya PHP script ile php artisan muzibu:migrate-songs
2
Database Path'lerini Güncelle

Database'deki file_path field'ını yeni path formatına güncelle.

// Migration veya Tinker ile DB::table('muzibu_songs')->update([ 'file_path' => DB::raw("CONCAT('muzibu/songs/original/song-', id, '.mp3')") ]); // Veya loop ile Song::chunk(1000, function ($songs) { foreach ($songs as $song) { $song->file_path = "muzibu/songs/original/song-{$song->id}.mp3"; $song->save(); } });
3
Metadata Çıkarma (Opsiyonel ama Tavsiye)

Tüm şarkıların süresini, bitrate'ini vs. database'e kaydet.

// getID3 kullanarak use getID3; Song::chunk(500, function ($songs) { $getID3 = new getID3; foreach ($songs as $song) { $path = Storage::disk('public')->path($song->file_path); $info = $getID3->analyze($path); $song->duration = $info['playtime_seconds'] ?? null; $song->bitrate = $info['audio']['bitrate'] / 1000 ?? null; $song->save(); } });
4
Lazy HLS Conversion Sistemi Kur

Şarkılar ilk çalındığında HLS'e dönüştürülecek (hepsini birden değil!).

Detaylar bir sonraki bölümde...

Süre Tahmini
  • Adım 1 (Kopyalama): 25,000 şarkı × 5MB ortalama = ~125GB → 1-2 saat (disk hızına bağlı)
  • Adım 2 (Path güncelleme): 25,000 satır → 5-10 dakika
  • Adım 3 (Metadata): 25,000 şarkı × 0.1 saniye = ~40 dakika (chunk chunk yapılır)
  • Toplam: 2-3 saat (tek seferlik iş)
KARAR #4: Lazy HLS Conversion

Şarkılar HLS'e Ne Zaman Dönüştürülecek?

İLK ÇALINDIĞINDA! (Lazy Conversion)
Neden Tüm Şarkıları Birden Çevirmiyoruz?
  • 25,000 şarkıyı birden çevirmek:
    • 125GB × 2 = 250GB disk kullanır (orijinal + HLS)
    • FFmpeg ile 25,000 şarkı × 10 saniye = ~70 saat CPU
    • Sunucu çöker, site yavaşlar
  • Lazy conversion:
    • Sadece çalınan şarkılar dönüştürülür
    • İlk kullanıcı 5-10 saniye bekler (bir kez)
    • Sonraki tüm kullanıcılar hızlı stream eder
    • Disk kullanımı kademeli artar
Lazy Conversion Akışı
// Frontend: Kullanıcı şarkıyı çalmak istiyor function playSong(songId) { fetch(`/api/songs/${songId}/stream`) .then(response => response.json()) .then(data => { if (data.status === 'converting') { // Göster: "Şarkı hazırlanıyor, lütfen bekleyin..." setTimeout(() => playSong(songId), 3000); // 3 saniye sonra tekrar dene } else { // HLS player'a URL ver player.loadSource(data.hls_url); player.play(); } }); }
// Backend: /api/songs/{id}/stream endpoint public function stream(Song $song) { // Eğer HLS'e dönüştürülmüşse if ($song->hls_converted) { return response()->json([ 'status' => 'ready', 'hls_url' => Storage::disk('public')->url($song->hls_path) ]); } // Henüz dönüştürülmemişse, job'a at if (!Cache::has("song-converting-{$song->id}")) { Cache::put("song-converting-{$song->id}", true, 600); // 10 dakika ConvertToHLSJob::dispatch($song); } return response()->json([ 'status' => 'converting', 'message' => 'Şarkı hazırlanıyor, lütfen bekleyin...' ]); }
// Job: ConvertToHLSJob.php public function handle() { $inputPath = Storage::disk('public')->path($this->song->file_path); $outputDir = "muzibu/songs/hls/song-{$this->song->id}"; $outputPath = Storage::disk('public')->path($outputDir); // HLS'e dönüştür (FFmpeg) $cmd = "ffmpeg -i {$inputPath} -codec: copy -start_number 0 " . "-hls_time 10 -hls_list_size 0 -f hls {$outputPath}/playlist.m3u8"; exec($cmd); // Database güncelle $this->song->update([ 'hls_path' => "{$outputDir}/playlist.m3u8", 'hls_converted' => true ]); Cache::forget("song-converting-{$this->song->id}"); }
Önemli Notlar
  • FFmpeg gerekli: Sunucuda FFmpeg kurulu olmalı
  • Queue sistemi: Laravel queue worker çalışmalı (php artisan queue:work)
  • İlk çalma yavaş: Kullanıcı 5-10 saniye bekleyebilir
  • Progress bar: Frontend'de "hazırlanıyor" animasyonu göster
KARAR #5: Görseller İçin Strateji

Görseller Nasıl Yönetilecek?

STRING PATH + THUMBMAKER (Zaten var, mükemmel çalışıyor!)
Görseller İçin Adımlar
1
Eski Görselleri Yeni Dizine Kopyala
cp /old/images/albums/* storage/app/public/muzibu/images/albums/ cp /old/images/artists/* storage/app/public/muzibu/images/artists/
2
Database'de thumb Field'ını Güncelle
// Albums Album::chunk(1000, function ($albums) { foreach ($albums as $album) { $album->thumb = "muzibu/images/albums/album-{$album->id}.jpg"; $album->save(); } }); // Artists Artist::chunk(1000, function ($artists) { foreach ($artists as $artist) { $artist->thumb = "muzibu/images/artists/artist-{$artist->id}.jpg"; $artist->save(); } });
3
Blade/Livewire'da Thumbmaker Kullan
{{-- Album kapağı göster (400x400 WebP) --}} <img src="{{ thumbmaker($album->thumb, ['w' => 400, 'h' => 400]) }}" alt="{{ $album->title }}" loading="lazy"> {{-- Sanatçı fotoğrafı (yuvarlak thumbnail) --}} <img src="{{ thumbmaker($artist->thumb, ['w' => 200, 'h' => 200, 's' => 1]) }}" class="rounded-circle"> {{-- Thumbmaker otomatik: - WebP'ye çevirir (küçük dosya) - Cache'ler (bir kez oluşturur) - Optimize eder (kalite 85) --}}
Neden Thumbmaker Mükemmel?
  • On-the-fly: İlk gösterimde thumbnail oluşturur
  • Cache: Bir kez oluşturur, sonra hızlı
  • WebP: Otomatik optimize (küçük dosya boyutu)
  • Esneklik: İstediğin boyut/kalitede kullan
  • Zaten var: Sisteminizde çalışıyor, yeni kod yok!

  7. Frontend player: HLS.js entegrasyonu yap
